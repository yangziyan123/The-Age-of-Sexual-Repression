# 峰哥亡命天涯语录网站设计方案（Flask）

## 1. 项目目标与基调
- 目标：将“峰哥亡命天涯语录”从纯文本素材升级为可浏览、可检索、可沉浸阅读的网页档案。
- 核心体验：像翻阅一部时代见证录，既有“历史厚重感”，又有“哲学思辨感”。
- 设计关键词：史料感、克制、留白、层次、沉浸、可回味。

## 2. 风格方向（历史厚重 + 哲学感）

### 2.1 视觉氛围
- 主色建议：
  - 背景：旧纸色 `#EAE2D2`、深夜墨色 `#171513`
  - 主文字：炭黑 `#23201C`
  - 强调色：铜金 `#8B6A3C`、暗红 `#6D2B2B`
- 质感设计：
  - 使用轻微纸张纹理、颗粒噪点、边缘暗角，制造“档案馆”观感。
  - 局部使用“版画风”线条分割或印章元素，强化历史语境。

### 2.2 字体策略
- 中文正文：优先宋体气质（思源宋体/霞鹜文楷等可商用字体）。
- 中文标题：可用更具碑刻感或黑体衬线对比，形成历史与现代冲突。
- 英文辅助：衬线字体（如 Cormorant Garamond / EB Garamond）提升哲学感。
- 排版原则：
  - 行宽控制在 32-42 个汉字可读区间。
  - 章节首段可用放大首字（Drop Cap）增强文献感。

### 2.3 图形语言
- 模块边框：细线框 + 轻微磨损效果。
- 图标体系：线性、克制，不使用卡通或扁平化大色块。
- 背景装饰：地图纹理、旧报纸拼贴、时间刻度线。

## 3. 信息架构（IA）

### 3.1 页面结构建议
1. 首页（沉浸入口）
- 大标题 + 副标语（例如“时代之下，人之困境”）。
- 每日/随机语录卡（可刷新）。
- 主题入口（生存、命运、金钱、关系、时代、自省）。
- 时间轴入口。

2. 语录总览页
- 支持按主题、年份、情绪标签筛选。
- 支持关键词搜索（全文检索）。
- 卡片列表 + 摘要句。

3. 语录详情页
- 主语录（大字号展示）。
- 上下文（来源、时间、场景说明）。
- 相关语录推荐（同主题/同情绪）。
- “哲学注脚”区块（可选手工撰写简评）。

4. 时间轴页
- 以年份/阶段分段，展示语录随时间的思想变化。
- 滚动触发渐显，突出“历程感”。

5. 关于页
- 项目缘起、收录原则、引用规范。

### 3.2 内容模型建议
每条语录建议字段：
- `id`
- `text`（原文）
- `author`（默认“峰哥”）
- `source`（出处）
- `created_at`（原始时间，可空）
- `era`（阶段标签）
- `topics`（主题标签）
- `mood`（情绪标签）
- `commentary`（注解）
- `is_featured`（是否精选）

## 4. 交互与动效（丝滑但克制）

### 4.1 动效原则
- 慢入快出、低幅度、短路径，避免花哨。
- 页面主过渡时长 300-600ms，滚动触发 200-400ms。
- 动画服务叙事：让用户“进入历史现场”，而非打断阅读。

### 4.2 动效场景
- 首屏开场：标题和副标题错峰淡入，背景纹理缓慢浮动。
- 语录卡悬停：轻微抬升 + 阴影加深 + 边框高亮。
- 时间轴滚动：节点按时间顺序逐个显现，配合细线延展。
- 详情页切换：语录正文交叉淡入，避免突兀跳变。

### 4.3 技术实现建议
- 纯 CSS + 少量 JS（Intersection Observer）即可覆盖大部分动效。
- 动效库可选 GSAP（复杂时间轴时）或 AOS（快速搭建时）。
- 为 `prefers-reduced-motion` 提供降级，保障可访问性。

## 5. Flask 技术架构设计

### 5.1 推荐目录结构
```text
project/
  app/
    __init__.py
    routes/
      main.py
      quote.py
    templates/
      base.html
      index.html
      quotes.html
      quote_detail.html
      timeline.html
      about.html
      components/
        quote_card.html
        timeline_item.html
    static/
      css/
        variables.css
        base.css
        layout.css
        components.css
        animations.css
      js/
        main.js
        timeline.js
      img/
        textures/
        icons/
    models/
      quote.py
    services/
      quote_service.py
      search_service.py
    data/
      quotes.json
  migrations/
  config.py
  run.py
  requirements.txt
```

### 5.2 模块职责
- `routes/`：按页面和资源拆分路由，保持单文件清晰。
- `services/`：业务逻辑（筛选、推荐、搜索）下沉，避免路由臃肿。
- `models/`：若后续引入数据库（SQLite/PostgreSQL），可平滑迁移。
- `templates/components`：可复用组件模板，避免重复 HTML。

### 5.3 渲染策略
- 以服务端渲染（SSR）为主，保证首屏可读与 SEO。
- 局部交互（筛选、加载更多）使用轻量 JS 增强。
- 若后期内容量增大，可为搜索和筛选提供 JSON API。

## 6. 数据与检索方案

### 6.1 起步阶段（轻量）
- 数据源：`quotes.json`。
- 索引策略：应用启动时载入内存，按 `topics/mood/era` 建立字典索引。
- 优点：实现快、部署简单。

### 6.2 成长阶段（可扩展）
- 存储迁移至 SQLite + SQLAlchemy。
- 全文检索可选：
  - SQLite FTS5（中小规模足够）
  - Whoosh/OpenSearch（大规模检索）

## 7. 页面级设计细化

### 7.1 首页
- Hero 区：大标题 + 历史照片纹理背景 + 一句核心语录。
- 今日语录：显示日期与语录，营造“每日翻阅档案”体验。
- 主题入口：六宫格或横向卷轴。
- 底部：项目说明与最近更新。

### 7.2 语录列表页
- 左侧（桌面端）：筛选栏（主题、年代、情绪）。
- 右侧：语录卡流。
- 移动端：筛选收纳为抽屉层，保证阅读面积。

### 7.3 详情页
- 主语录居中大排版。
- 下方补充出处、注解、相关推荐。
- 加入“上一篇/下一篇”提升连续阅读感。

## 8. 性能与工程质量
- 图片纹理压缩（WebP/AVIF），首屏关键纹理预加载。
- CSS 分层：变量、基础、组件、动画拆分，便于维护。
- 延迟加载非关键 JS。
- 建议增加基础测试：
  - 路由可用性测试（Flask test client）
  - 数据字段完整性测试

## 9. 可访问性与国际化预留
- 颜色对比度达到 WCAG AA。
- 所有关键交互支持键盘操作。
- 页面语义标签完整（`main`、`article`、`section`）。
- 文案与标签统一通过配置管理，后续可扩展英文界面。

## 10. 分阶段实施计划
1. 第一阶段（1-2 天）
- Flask 脚手架与基础模板搭建。
- 首页 + 列表页静态版（含基础风格变量）。

2. 第二阶段（2-3 天）
- 数据模型落地（JSON/SQLite 二选一）。
- 完成筛选、搜索、详情页、相关推荐。

3. 第三阶段（1-2 天）
- 时间轴页 + 动效细化。
- 性能优化与可访问性修正。

4. 第四阶段（持续）
- 补充语录注解体系。
- 增加管理后台（可选，Flask-Admin 或自建）。

## 11. 设计验收标准
- 视觉：能明显感受到“历史厚重 + 哲学沉思”的统一气质。
- 交互：动画丝滑、无卡顿、无突兀闪烁。
- 内容：语录检索和分类高效，详情页阅读沉浸。
- 工程：结构清晰，后续扩展到数据库与后台成本可控。
